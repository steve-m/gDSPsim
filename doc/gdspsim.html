gDSPsim Documentation

This is a DSP simulator for the TMS320C54x DSP chip from Texas Instruments written to run on Linux. There is work to extend the simulator to the TMS320C55x DSP chip. The software uses gtk, gnome, and Gtk+Extra. Note: in the .rpm file, Gtk+Extra has been statically linked for convenience.

Support Software
Assembler: You can download an assembler for the C54x from here When configuring it, be sure to specify the target: ./configure --target=c54x

Command line options
-f [filename] Read in [filename]

Shortcut Keys
[F5] Runs the simulation
[F6] Stops the simulation
[F7] Animate the simulation (Runs slower, with lots of highlighting)
[F8] Steps through the code 1 opcode at a time.

Decode Window
Setting Break points: In the decode window click in the left column to set a break point and a stop sign will appear when it is properly set. To remove the break point click on the stop sign and the stop sign will disappear confirming the removal of the break point.
When stepping through code, color bars will track the code where each color represents a different stage of the pipeline. You can set how stepping through the code tracks the color highlighting in the preferences for the window.
The 2 entries at the top of the window set the memory addresses that are decoded in the window. You can enter hex numbers or label names in the entry block

Register Window
When registers are changed the turn red. If the value changes while running, then changes back to the original value, the value stays black. This may change in the future.

Memory Window
The location of the stack pointer is highlighted yellow. When values are changed they are changed to red. When values are read, they are turned light-blue. The color are maintained even if the value changes back to it's original value between break points. In the future the number of times a memory location is read or written too may be available. The 2 entries at the top of the window set the memory addresses that are shown in the window. You can enter hex numbers or label names in the entry block. The Data/Program button at the top toggles the type of memory displayed.

File IO
The input file should just be a column of numbers in either decimal or hexidecimal (0x0) format.

Known Bugs
Plotting: When changing the plot, the axis don't update properly until you grab the plot with the mouse and move it.
